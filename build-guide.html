How to build the whole webpage

A beginner-friendly file-by-file guide using plain HTML, CSS, images, and GitHub Pages.

Folder structure

Create a project folder with this exact structure:

b2-spirit-site/ ├── index.html ├── fabrication.html ├── repair.html ├── gallery.html ├── build-guide.html ├── style.css ├── B2_Spirit_Fabrication_Lab_Memo_Revised.docx └── assets/ ├── final-gloss.jpg ├── layers-sketch.png ├── matte-finish.jpg ├── peel-ply-layer.jpg ├── post-bottom-cleanup-1.jpg ├── post-bottom-cleanup-2.jpg ├── project-selfie.jpg ├── repair-closeup.jpg ├── repair-full.jpg ├── two-models.jpg ├── vacuum-blue-bag.jpg └── vacuum-pump.jpg

Step 1: Make the files

  1. Create a new folder on your computer.
  2. Inside it, make an assets folder.
  3. Create the five HTML files listed above.
  4. Create one stylesheet named style.css.
  5. Put all of your project JPG and PNG files inside the assets folder.

Step 2: Build the home page

Start with index.html. Every HTML page needs the same basic pieces: a head section, a title, a link to the CSS file, a header with navigation, and a main content area.

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>B-2 Spirit Composite Model</title> <link rel="stylesheet" href="style.css"> </head> <body> <header>...navigation...</header> <main>...content...</main> </body> </html>

After the page skeleton works, add the hero image, overview paragraph, and acknowledgment section.

Step 3: Reuse the same navigation on every page

Copy the same navigation bar into every HTML file. The only thing you change is which link gets the active class.

<nav> <ul> <li><a href="index.html">Overview</a></li> <li><a href="fabrication.html">Fabrication</a></li> <li><a href="repair.html">Repair & Spares</a></li> <li><a href="gallery.html">Gallery</a></li> <li><a href="build-guide.html">Build Guide</a></li> </ul> </nav>

Step 4: Add images correctly

Put every image in the assets folder and point to it with a relative path. For example:

<img src="assets/final-gloss.jpg" alt="Completed glossy B-2 Spirit carbon fiber model">

Keep the file names simple and do not use spaces. Hyphens are easier to manage.

Step 5: Style everything in one CSS file

Put all colors, spacing, fonts, image sizing, and layout rules inside style.css. That way, one edit changes the whole website.

This site uses one shared stylesheet so every page looks consistent.

Step 6: Test locally

  1. Double-click index.html and open it in your browser.
  2. Click every navigation link and make sure every page loads.
  3. Check whether every image appears.
  4. Fix missing images by checking spelling and folder paths.

Step 7: Upload to GitHub Pages

  1. Create a new GitHub repository.
  2. Upload the full website folder contents so that index.html is in the repository root.
  3. Upload the assets folder too.
  4. In repository settings, open the Pages section and publish from the main branch root folder.
  5. Wait for GitHub to deploy the site, then open the provided website URL.
